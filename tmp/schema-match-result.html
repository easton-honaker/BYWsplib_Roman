Ingestion Notes

  • Photometry fan-out: Each source row in the CSV expands to up to 9 Photometry rows (i, z, y, W1, W2, Y, J, H, K) — only insert rows where the magnitude is non-null.
  • NIR band SVO ID resolution: The NIR ref string encodes per-band survey sources. The ingestion script must parse this string to assign the correct SVO filter ID per band. "2MASS reject" values should be excluded. Bands from different surveys in the same source will generate separate Photometry rows with different band values.
  • Dual spectral type rows: Both FINAL and "Spectral Type" map to SourceTypes. Ingest FINAL as adopted=True. Skip "Spectral Type" if it is identical to FINAL. Otherwise ingest as adopted=False with the same reference.
  • BΣ_YMGs multi-value parsing: Split on ";" to get individual association+probability tokens. Parse probability from parentheses (e.g. "ABDMG(88)" → association="ABDMG", membership_probability=0.88). Insert one Associations row per token per source.
  • Duplicate RA/Dec columns: Use catWISE RA/catWISE Dec for Sources. Skip RA SXG/DE SXG.
  • Disc column contains comma-separated values (e.g. "WB20, Azul") for some objects. Parse the primary reference (first token) for Sources.reference; remainder go to Sources.other_references.
  • Publications must be ingested first — all Disc, Spec, opt ref, NIR ref, plx_ref, PM_ref values must exist in Publications before any data table rows are inserted.
  • 9 objects with no Goodstuff data will have NULL for all photometry/astrometry columns — ingest Sources and SourceTypes only for those rows.
  • Spectra table reference: The Spec column should be used as Spectra.reference (the publication that describes the spectrum). Spectrum Instrument → Spectra.instrument (FK→Instruments); must add Instruments rows first.
